*Shared definitions for all terminology.*

This document provides a single source of truth for all acronyms and specialized terms used across the SAIFE Gateway documentation. Read this when you encounter an unfamiliar technical or legal term.

| Term / Abbreviation | Definition |
|---|---|
| **ADR** | Architecture Decision Record. A short document capturing an important architectural decision made along with its context and consequences. |
| **API** | Application Programming Interface. A set of rules and protocols for building and interacting with software applications. |
| **AVV** | Auftragsverarbeitungsvertrag (Data Processing Agreement). A legal contract required under the GDPR between a data controller and a data processor. |
| **CI** | Continuous Integration. The practice of automating the integration of code changes from multiple contributors into a single software project. |
| **DDoS** | Distributed Denial of Service. A malicious attempt to disrupt normal traffic by overwhelming the target with a flood of Internet traffic. |
| **DPF** | Data Privacy Framework. An agreement facilitating cross-border data transfers. |
| **DPIA** | Data Protection Impact Assessment. A process designed to describe the processing of data, assess its necessity and proportionality, and help manage the risks to the rights and freedoms of natural persons. |
| **DPO** | Data Protection Officer. An enterprise security leadership role required by the GDPR. |
| **DSFA** | Datenschutz-Folgenabschätzung. The German equivalent term for a Data Protection Impact Assessment (DPIA). |
| **DSL** | Domain-Specific Language. A computer language specialized to a particular application domain, such as the rule definition format for educational guardrails. |
| **FRIA** | Fundamental Rights Impact Assessment. An assessment required by the EU AI Act to evaluate the impact of an AI system on fundamental rights. |
| **GDPR** | General Data Protection Regulation. The primary data protection and privacy regulation in the European Union. |
| **HMAC** | Hash-based Message Authentication Code. A specific type of message authentication code involving a cryptographic hash function and a secret cryptographic key. |
| **JWT** | JSON Web Token. A proposed Internet standard for creating data with optional signature and/or optional encryption whose payload holds JSON that asserts some number of claims. |
| **LDP** | Local Differential Privacy. A privacy framework that allows data to be collected and analyzed while injecting statistical noise locally, guaranteeing that an individual's data cannot be distinguished. |
| **LLM** | Large Language Model. A very large machine learning model trained to understand and generate human language. |
| **LMS** | Learning Management System. A software application for the administration, documentation, tracking, reporting, automation, and delivery of educational courses. |
| **ML** | Machine Learning. A field of artificial intelligence focused on building systems that learn from data. |
| **mTLS** | Mutual Transport Layer Security. A security protocol that requires both the client and the server to authenticate each other using digital certificates, ensuring secure bidirectional communication. |
| **PII** | Personally Identifiable Information. Any data that could potentially identify a specific individual. |
| **POC** | Proof of Concept. A realization of a certain method or idea to demonstrate its feasibility. |
| **PR** | Pull Request. A method of submitting contributions to an open-source project. |
| **ReDoS** | Regular Expression Denial of Service. An algorithmic complexity attack that exploits the backtracking mechanism of regular expression engines to exhaust CPU resources. |
| **SAIFE** | Secure AI For Education. The core project name for this educational safety middleware. |
| **SCC** | Standard Contractual Clauses. Standardized model data protection clauses approved by the European Commission. |
| **TIA** | Transfer Impact Assessment. An assessment required when transferring personal data to third countries outside the EU/EEA. |
| **TOM** | Technical and Organizational Measure. Security measures implemented to ensure a level of security appropriate to the risk of data processing under the GDPR. |
| **TTL** | Time to Live. A mechanism that limits the lifespan or lifetime of data in a computer or network. |
| **XML** | Extensible Markup Language. A markup language that defines a set of rules for encoding documents in a format that is both human-readable and machine-readable. |
